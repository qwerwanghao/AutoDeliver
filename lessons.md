## 技术决策记录

### 架构决策

**决策**: 采用五阶段工作流
**理由**:
1. 将复杂项目分解为可管理的阶段
2. 每个阶段有明确的输入和输出
3. 支持阶段间暂停和恢复
4. 便于质量控制和用户确认

**决策**: 使用 Memory 持久化
**理由**:
1. AI 上下文窗口有限
2. 需要跨阶段传递大量信息
3. 支持断点续传
4. 便于项目复盘和学习

**决策**: 分离发现和规划阶段
**理由**:
1. 发现阶段专注于理解和分析
2. 规划阶段专注于设计和分解
3. 避免混合导致混乱

## 代码约定

### Bash 脚本约定

- 使用 `set -euo pipefail` 严格模式
- 函数使用 `log_info`, `log_error`, `log_success` 统一日志
- 阶段脚本返回 0 表示成功，非 0 表示失败
- 所有环境变量导出供子脚本使用

### 文件组织约定

- `context/` - 用户输入（只读）
- `memory/` - AI 工作记忆（读写）
- `deliver/` - 最终交付物（只写）
- `workflows/` - 工作流脚本
- `prompts/` - 提示词模板

## 常见问题

### Q1: 如何从现有项目开始？

**A**: 使用 `-p` 参数指定项目路径：

```bash
ai-deliver run -p /path/to/existing/project
```

### Q2: 如何从特定阶段恢复？

**A**: 使用 `-s` 参数：

```bash
ai-deliver run -s plan    # 从规划阶段开始
ai-deliver run -s build   # 从构建阶段开始
```

### Q3: 如何跳过用户确认？

**A**: 使用 `-y` 参数：

```bash
ai-deliver run -y
```

### Q4: 如何查看项目状态？

**A**: 使用 status 命令：

```bash
ai-deliver status
```
